Contexte et Comparaison

Un peu de contexte dans la cadre de la Transformation Digitale

Les approches de modélisation - Event Modeling, Domain Storytelling and Event Storming - ont pour objectif de creuser les connaissances métier, collaborativement, de manière multidisciplinaire, visuelle, interactive et détaillée

Ces approches s'intégrent dans les phases d'analyse de processiong AS-IS & TO-BE et d'implémentation/Expérimentation

Et si nous comparons ces approches avec BPMN et ArchiMate, elles se différencient par leur simplicité, accessibilité et "interactivité".

Comparaison des 3 approches


    Domain Storytelling

  • Domain Storytelling est extrêment polyvalent et egon.io (outil électronique) est efficace, que vous soyez en présentiel ou "remote"
  • Domain Storytelling est le couteau Suisse - toujours dans la poche, dès qu'on parle d'interactions.

    • Event Storming

  • Event Storming est plutôt une approche en présentiel (côté Storming): il reste assez polyvalent
  • Event Storming peut être combiné avec avec Event Modeling.

    • Event Modeling

  • Event Modeling, très agréable au format électronique, il est très très efficace dans la phase projet
  • Event Modeling une ferrari pour les projets de développement ou légacy.

  • C'est mon expérience donc cela reste une opinion qui peut changer rapidement
